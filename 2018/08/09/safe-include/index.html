文件包含漏洞小结

Posted on 八月 09,2018 | In Web安全

文件包含

包含操作是指大多数语言提供的功能
常见如下:

包含漏洞在php中经常出现，但并不意味着只在php中出现

文件包含漏洞成因

代码重用时经常进行文件包含，如果参数暴露可控
通过恶意操作，可以达到攻击的目的

文件包含相关函数

  • include
  • include_once
  • require
  • require_once

前两者遇到错误会继续,后两者会报错退出

文件包含类型

LFI 本地文件包含

RFI 远程文件包含

要求

  • allow_url_include = ON
  • allow_url_fopen = ON

包含文件并解析成PHP，如果fopen为OFF，需要伪协议才能包含

文件包含截断技术

开发者常用这样的语句进行文件类型的限定

导致无法包含指定文件

%00截断

路径长度截断

  • Windows下目录最大长度为256字节，超出的部分会被丢弃；
  • Linux下目录最大长度为4096字节，超出的部分会被丢弃。

‘?’问号伪截断 RFI

#号空格截断

php://input截断

附录:PHP伪协议

php://filter（本地磁盘文件进行读取）

php://input(post)

php://input（写shell）

三个条件

  • allow_url_fopen on
  • allow_url_include on
  • 满足php<5.3.0

php://input（命令执行）

file://伪协议（读取文件内容）

data://伪协议

data://(读取文件)

phar://伪协议

  • PHP >= 5.3.0
  • 压缩包需要是zip协议压缩

zip://伪协议

  • PHP >= 5.3.0

注意在windows下测试要5.3.0<PHP<5.4 才可以 #在浏览器中要编码为%23，否则浏览器默认不会传输特殊字符。
